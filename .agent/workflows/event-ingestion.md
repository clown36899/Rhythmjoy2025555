# 이벤트 정밀 수집 워크플로우 (Event Ingestion Recipe)

이 문서 워크플로우의 목적은 아래 조건들에 명시된 이벤트를 조회수집하는것이다.

**이 문서는 사용자의 명시적 동의 하에 수정할수있다**
**지정된 keyword(스윙스캔들,스윙프렌즈,해피홀,스윙타임,경성홀,박쥐스윙,대전스윙피버,스윙홀릭 | `dj`, `소셜`)**

**오늘 = 한국시간으로 지금이다**



> [!CAUTION]
> **에이전트 실행 강제 수칙 (Enforcement Rules)**
> 0. **수집 대상 유형 필터 [최최우선]**: 수집 대상은 **DJ 소셜 파티**만 해당한다. 다음은 **절대 수집하지 않는다**:
>    - ❌ 정규 강습/클래스 (입문반, 초급, 중급, 상급 등 모든 레벨)
>    - ❌ 워크숍 (단기, 집중, 원데이 등 모든 형태)
>    - ❌ 공연/발표회/대회
>    - ❌ 후기/리뷰/회고 게시물
>    - ✅ 수집 대상: DJ가 참여하는 소셜 댄스 파티, 정기 소셜, 라이브 파티
>    - **판별 기준**: 포스터/캡션에 "소셜", "Social", "파티", "Party", "DJ" 키워드 중 하나 이상 포함
> 1. **지시문(Task) 원문 주입**: 하위 에이전트(`browser_subagent`)에게 업무를 지시할 때, 반드시 가이드의 핵심 원칙을 **토씨 하나 틀리지 않고 복사하여 명령문에 포함**해야 합니다. (예: "스크린샷은 반드시 포스터 영역만 정밀 크롭할 것")
> 2. **`task.md` 체크리스트 구체화 의무**: task.md의 각 소스 항목에는 가이드의 구체적인 URL, 검색어 텍스트를 그대로 포함해야 한다. 단순화하지 않는다.
>    - ❌ **금지 표현**: `구글 검색 및 대상 URL 확보`, `키워드 검색`, `수집 준비 및 키워드 확인`, `네이버 검색 및 대상 URL 확보`
>    - ✅ **필수 표현**: `구글 검색: 사보이볼룸 소셜 dj 인스타그램 (도구>지난 이번주)`, `네이버 접속: https://cafe.naver.com/f-e/cafes/14933600/menus/501?viewType=I`, `인스타그램 접속: https://www.instagram.com/happyhall2004/`
> 3. 로그인하지 않는다. 로그인 때문에 조회가 안될경우 조회를 포기하고 다음 스탭으로 넘어간다.
> 4. 검색이 필요할경우 지정된 키워드 만 사용하고 확장하지않는다.
> 5. **task.md 템플릿 사용 의무화**: task.md 작성 시 반드시 본 가이드 하단의 **[필수 task.md 템플릿](#필수-taskmd-템플릿)**을 복사하여 사용할 것. 소스를 뭉뚱그려 '키워드 검색', '구글 검색' 등으로 표기하는 것을 **절대 금지**한다.
> 6. **"결과 없음 = 정상 완료" 원칙 [최우선]**: 지정된 소스에서 해당 기간의 게시물이 발견되지 않는 것은 **실패가 아닌 정상적인 결과**이다. `판정: 결과 없음 → 스킵`으로 기록하고 **즉시** 다음 소스로 이동한다. 추가 검색, 다른 경로 탐색, 검색어 변형은 **절대 금지**한다.
> 7. **"데이터 갭" 해석 금지 [최우선]**: 특정 날짜의 이벤트 데이터가 없을 때, 이를 "data gap", "비정상적 누락", "조사 필요한 불일치" 등으로 해석하는 것을 **절대 금지**한다. 게시물이 아직 올라오지 않았을 뿐이며, 이는 완전히 정상이다. "아직 게시되지 않음"으로 기록하고 **즉시 다음 소스로 이동**한다.
> 8. **소스별 종료 판정 기준**: 각 소스에서 확인할 범위는 다음으로 **한정**된다. 이 범위를 벗어나는 탐색은 금지:
>    - **구글 검색**: 필터 적용 후 검색 결과 **1페이지**만 확인
>    - **네이버 카페**: 지정 URL 접속 후 **현재 보이는 오른쪽 리스트**만 확인
>    - **인스타그램**: 프로필 페이지에서 **최근 6개 포스트를 직접 클릭**하여 내용 확인 (썸네일만으로는 판단 불가 — 반드시 클릭하여 캡션/이미지 내용을 본다)
>    - ⛔ 금지 행위: 2페이지 탐색, 다른 메뉴/게시판 이동, 검색어 변형, 스크롤 다운하여 오래된 글 탐색, 필터 변경
> 9. **인스타그램 계정 접속 실패 시 즉시 스킵 [최우선]**: 지정된 인스타그램 URL에 접속했을 때 계정이 존재하지 않거나, 비공개이거나, 게시물이 로드되지 않으면 → **"계정 접속 불가 → 스킵" 기록 후 즉시 다음 소스로 이동**한다.
>    - ⛔ **절대 금지**: Google/네이버/Facebook에서 계정 검색, 핸들 추측(예: `tensocial.swing`, `ten_social_swing`), 해시태그 검색(`#tensocial`), 이미지 검색, 다른 SNS 플랫폼 탐색
> 10. **다른 계정 방문 절대 금지**: 지정된 소스의 인스타그램 계정 **이외의** 다른 사용자 프로필이나 포스트로 이동하는 것을 금지한다. (예: 경성홀 수집 중 `@joje_ellakim` 포스트 방문 금지)
> 11. **이미 확인/캡처한 포스트 재진입 금지**: 한 번 클릭하여 내용을 확인하거나 캡처를 완료한 포스트는 **다시 열지 않는다**. 필요한 정보는 첫 번째 진입 시 모두 수집한다.
> 12. **데이터 확보 후 추가 탐색 금지**: 필요한 이벤트 정보와 포스터 캡처를 완료했으면 **즉시 해당 소스를 종료**하고 다음 소스로 이동한다. "혹시 더 있을까" 하는 추가 확인은 6개 포스트 범위 안에서만 허용된다.
> 13. **인스타그램 확인 절차 (반드시 순서대로)**:
>    1. 프로필 페이지 접속
>    2. 최근 포스트 중 **1번째** 클릭 → 내용 확인 → 해당 기간 이벤트면 캡처+파싱, 아니면 닫기
>    3. **2~6번째** 포스트도 동일하게 순서대로 클릭하여 확인
>    4. 6개 확인 완료 또는 날짜가 수집 대상 기간보다 이전이면 → **즉시 종료**
>    5. 절대 7번째 이상 포스트를 확인하지 않는다

이 워크플로우는 소셜 미디어(인스타그램, 네이버 등)에서 이벤트 정보를 고화질로 수집하고 정보를 구조화하여 `EventIngestor` 시스템에 전달하는 표준 절차를 정의합니다.

## 0. 기존 이벤트 사전 확인 (Pre-check) [신규 필수 단계]
- 이벤트 수집을 시작하기 전에 현재 시스템(DB)에 이미 등록된 이벤트 목록을 반드시 먼저 확인해야 합니다.
- **실행 명령**: `node scripts/check-db-events.js [시작일(YYYY-MM-DD)] [종료일(YYYY-MM-DD)]`
  - 예시: `node scripts/check-db-events.js 2026-02-21 2026-03-10`
- 출력된 목록에서 자신이 조사할 소스(주최/장소/키워드)와 일치하는 **DJ 소셜 파티**가 특정 날짜에 이미 등록되어 있다면, **해당 소스의 리서치는 "이미 등록됨 → 스킵"으로 처리하고 건너뜁니다.** 이를 통해 불필요한 중복 수집 작업을 방지합니다.

    - **네이버**: '옵션 > 기간 > 1주일'을 선택합니다.


## 2. 정밀 수집 5단계 공정 (Precise Scraping Process)
단순 검색 결과 썸네일 수집 대신 아래의 정밀 공정을 준수해야 합니다.

0. **목록(Index) 선행 확보**: 게시판 진입 시, 수집 대상 기간(예: 2월 초)에 해당하는 게시물이 총 몇 건인지 목록을 먼저 추출하여 검증한다.
1. **이미지수집 [다운로드 우선 원칙]**: 구글 썸네일 캡처를 금지하고, 실제 게시물 URL로 직접 진입하여 고화질 이미지를 수집한다. **반드시 아래 순서를 따른다:**
    1. **(1순위) CDN 원본 다운로드**: JavaScript로 포스터 `img` 요소의 `src` URL(scontent CDN 주소)을 추출한 뒤, `curl -o` 명령으로 원본 이미지를 직접 다운로드한다. 이 방법이 가장 깨끗한 결과를 제공한다.
    2. **(2순위) 폴백 — 요소 기반 크롭 캡처**: CDN 다운로드가 실패(403, 만료 URL 등)한 경우에만, 포스터 이미지가 담긴 `img` 요소를 식별하여 `CaptureByElementIndex`로 정밀 크롭한다. 주변 UI가 포함된 전체 화면 스크린샷은 허용하지 않는다.
    - ⛔ **금지**: 전체 화면 스크린샷을 먼저 찍는 행위. 반드시 다운로드를 먼저 시도할 것.
2. **환경 정돈**: 진입 후 나타나는 로그인 권유 팝업이나 모달을 즉시 제거(Click x:659, y:250 등)하여 시야를 확보합니다.
4. **상세 정보 파싱**: 캡션 영역에서 아래 데이터를 구조화하여 추출합니다.
    - **행사 일자**: YYYY-MM-DD 형식 및 요일
    - **DJ 명단**: 출연 DJ들을 배열 형식으로 추출
    - **운영 시간**: 시작 및 종료 시간
    - **상태 및 가격**: 정상 운영 여부(Social Open, CLOSED 등) 및 입장료
    - **특이사항**: 설 연휴 휴무, 장소 변경 등 비고 항목
5. **파일 동기화**: 크롭된 이미지는 `public/scraped/[keyword]_poster.png`로 저장하고, 이미지 경로는 웹 경로(`/scraped/...`)로 기록합니다.

## 3. 데이터 업데이트 및 저장 (Data Sync)
- 수집된 정보를 `src/data/scraped_events.json` 형식에 맞춰 업데이트합니다.
- **필수 필드**: `id`, `keyword`, `poster_url`, `structured_data`, `source_url`.
- **중복 검사**: 수집된 데이터의 날짜와 제목이 기존 DB(`useEvents` 훅 참조)에 있는지 대조합니다.

## 4. 최종 검증 (Verification)
- 관리자 페이지 `http://localhost:5173/admin/ingestor`를 열어 아래 사항을 확인합니다.
    - 포스터 이미지가 찌그러짐 없이(`object-fit: contain`) 표시되는지.
    - 'CLOSED' 상태인 항목이 빨간색으로 시각화되는지.
    - 중복 항목에 'DUPLICATE' 배지가 정상 노출되는지.

## browser_subagent 탈선 방지 지시문 (필수 삽입)

> [!IMPORTANT]
> 하위 에이전트(`browser_subagent`)에게 작업 지시 시 반드시 아래 문구를 **그대로 복사**하여 Task에 포함할 것.

```
[탈선 방지 규칙 - 반드시 준수]
```
