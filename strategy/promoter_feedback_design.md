# 주최자 피드백 시스템 (Promoter Feedback System) 기획안

## 1. 개요
*   **목표**: 게시물 작성자(주최자)가 사이트에 방문했을 때, 자신의 게시물 성과를 직관적으로 확인하고 "내 홍보가 잘 되고 있구나"라는 효능감을 느끼게 함.
*   **전략**: V4 전략의 'C. 주최자를 춤추게 하라'의 구체적 실행.

## 2. 핵심 기능: "My Impact" 대시보드
메인 페이지 상단 또는 마이페이지 진입 시 가장 먼저 보이는 **개인화된 리포트 카드**.

### A. UI 구성 (카드 형태)
1.  **헤드라인 (Headline)**
    *   *"작가님의 글이 오늘 **156명**에게 도달했습니다!"*
    *   (성과가 좋을 때) *"축하합니다! 평소보다 **3배** 더 많은 관심을 받고 있어요! 🔥"*
    *   (성과가 저조할 때) *"조금만 더 힘내세요! 지금까지 **45명**이 확인했습니다."*

2.  **핵심 지표 (Key Metrics)**
    *   **누적 조회수**: `Total Views`
    *   **유입 기여도**: `Referral Count` (외부 공유 링크를 통해 들어온 숫자)
        *   *이게 핵심입니다. "내가 퍼다 날라서 이만큼 들어왔다"를 보여줘야 함.*
    *   **전환 클릭**: `Action Clicks` (신청 링크/오픈톡 클릭 수)

3.  **상대적 화력 (Relative Heat Index)**
    *   게이지 바(Bar) 형태
    *   `[ 내 글의 속도 ]` vs `[ 전체 평균 속도 ]`
    *   내 글이 평균보다 빠르면 "🚀 Hot" 배지 부착.

### B. 노출 위치
1.  **메인 배너 하단 (로그인 유저 전용)**
    *   주최자가 로그인하면 가장 잘 보이는 곳에 "내 글 현황"을 띄움.
2.  **게시글 상세 페이지 상단 (작성자 본인만 보임)**
    *   자기 글을 클릭해서 들어갔을 때, 관리자 도구처럼 상단에 통계 바(Stat Bar) 노출.

## 3. 구현 로직 (Logic)

### A. 데이터 집계
*   **대상**: 최근 7일 이내 작성한 게시물 중 가장 조회수가 높은 1개 (또는 합산).
*   **비교군**: 최근 30일간 작성된 유사 카테고리 게시물의 평균 조회수 속도.

### B. "유입 기여도" 추적 방법
*   게시물 공유 버튼 클릭 시 `?ref=share_user_id` 파라미터 자동 추가.
*   이 파라미터로 접속한 세션 수를 카운트하여 주최자에게 "당신의 공유 덕분"이라고 칭찬.

## 4. 기대 효과
*   **Lock-in**: 성과를 확인하고 싶어서 사이트에 더 자주 접속함 (통계 중독 유도).
*   **Viral**: "유입 기여도" 수치를 올리고 싶어서 자발적으로 더 열심히 링크를 공유함.
