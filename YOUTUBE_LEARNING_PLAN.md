# [기획서] 댄스 학습 갤러리 (YouTube Learning Gallery) 구축 계획 안내

사용자님의 요청에 따라 수집하신 유튜브 재생목록과 즐겨찾기를 다른 사용자들에게 교육 및 학습용으로 공유하기 위한 **독립적인 갤러리 프로젝트** 기획서입니다.

---

## 1. 프로젝트 개요 (Overview)

- **프로젝트명**: 댄스 학습 갤러리 (Code Name: `LearningLayout`)
- **목적**: 관리자가 큐레이션한 양질의 댄스 영상 및 재생목록을 사용자가 쉽고 체계적으로 모아볼 수 있는 학습 전용 공간 제공
- **핵심 가치**: "유튜브의 방대한 자료를 '요리'하여 떠먹여 주는 친절한 학습 가이드"
- **특징**:
    - 기존 빌보드/소셜과는 완전히 분리된 **독립 라우트** (`/learning`)
    - 영상 파일 직접 저장 없음 (YouTube Embed 활용)
    - 관리자 중심의 콘텐츠 큐레이션 (목록 생성, 편집, 배치)

## 2. 주요 기능 (Key Features)

### 2.1. 사용자 기능 (User Side)
1.  **갤러리 홈 (`/learning`)**
    - **카드형 레이아웃**: 재생목록(커리큘럼) 단위로 썸네일과 제목이 카드 형태로 나열
    - **카테고리 필터**: 장르별(Lindy Hop, Solo Jazz, Balboa 등) 또는 난이도별 분류
    - **검색 기능**: 제목, 태그 기반 검색

2.  **재생목록 상세 뷰 (Playlist Detail)**
    - **학습 모드 플레이어**: 상단에 메인 플레이어, 하단/측면에 영상 리스트 배치
    - **연속 재생**: 한 영상이 끝나면 다음 영상으로 자동 전환
    - **진도율 체크(Optional)**: "봤어요" 체크 기능 (로컬 스토리지 활용하여 가볍게 구현)

### 2.2. 관리자 기능 (Admin Side)
1.  **콘텐츠 관리 시스템 (CMS)**
    - **유튜브 링크 파싱**: 재생목록 URL 또는 영상 URL 입력 시 메타데이터(제목, 썸네일) 자동 추출
    - **커스텀 목차 구성**: 유튜브 상의 순서와 상관없이 관리자가 원하는 순서로 재배치 가능
    - **설명 및 태그 추가**: 각 영상이나 목록에 학습 포인트(Tip) 메모 작성 기능

## 3. 시스템 아키텍처 (System Architecture)

### 3.1. 라우팅 구조
기존 `App.tsx` 라우팅에 독립적인 경로를 추가합니다.

```
/learning          (메인 갤러리/홈)
/learning/:listId  (상세 학습 뷰/플레이어)
```

### 3.2. 데이터베이스 설계 (Supabase)
새로운 테이블 2개를 신설하여 유기적으로 관리합니다.

**1) `learning_playlists` (학습 코스/목록)**
- `id`: UUID
- `title`: 목록 제목 (예: "초급 찰스턴 마스터하기")
- `description`: 설명
- `thumbnail_url`: 대표 썸네일
- `category`: 장르/분류
- `tags`: 태그 배열
- `is_public`: 공개 여부
- `created_at` / `updated_at`

**2) `learning_videos` (개별 영상)**
- `id`: UUID
- `playlist_id`: 소속된 목록 ID (FK)
- `youtube_video_id`: 유튜브 영상 식별자 (예: `dQw4w9WgXcQ`)
- `title`: 영상 제목 (관리자가 수정 가능)
- `order_index`: 재생 순서
- `memo`: 관리자 코멘트/팁

### 3.3. 기술 스택
- **Frontend**: React, React Router, YouTube IFrame API (기존 `billboard` 로직 재사용 및 경량화)
- **Styling**: Vanilla CSS (기존 디자인 시스템 계승하되, 학습에 집중할 수 있는 깔끔한 테마 적용)
- **Backend**: Supabase (PostgreSQL)
