# 타장르 씬 수집 설계

**작성일**: 2026-05-26  
**대상 범위**: 살사, 바차타, 탱고, 스트릿  
**제외 범위**: 현대무용, 고전무용, 일반 K-pop 커버/상시 클래스, BAT SWING

## 목적

스윙 자동 수집은 안정 수집 루트로 유지하고, 타장르는 곧바로 운영 후보로 넣지 않는다. 먼저 장르별로 씬이 어디에서 움직이는지, 어떤 계정/사이트가 실제 일정 원본인지, 이미지와 날짜를 안정적으로 얻을 수 있는지 확인한 뒤 검증된 소스만 저장 단계로 승격한다.

최종 목표는 단순한 이벤트 목록이 아니라 **특정 장르를 공유하는 씬의 지도(scene map)** 를 만드는 것이다. 수집은 그 지도를 그리기 위한 증거 수집이어야 한다. 즉, 강습/소셜/행사/모집을 몇 개 찾았는지보다 “그 장르가 어떤 방식으로 운영되고, 사람들이 어떻게 유입되고, 어디서 돈이 돌고, 어떤 공간과 조직이 중심인지”를 설명할 수 있어야 한다.

## 씬 맵 산출 관점

타장르 조사는 아래 질문에 답할 수 있도록 자료를 모은다.

| 축 | 확인할 내용 | 증거 예시 |
|---|---|---|
| 활동 비중 | 강습, 소셜, 행사, 모집이 어느 정도 비중인지 | 최근 일정 수, 반복 빈도, 공식 캘린더 |
| 현재 인구/활동량 | 씬이 활발한지, 참여자가 어느 정도인지 | 소셜 주기, 댓글/참여 안내, 정원/마감 공지, 반복 운영 기간 |
| 참여 경로 | 초보자가 어떻게 들어오는지 | 입문반, 오픈클래스, 체험회, 소셜 입장 조건 |
| 구성 방식 | 학원 중심인지, 바/공간 중심인지, 동호회/크루 중심인지 | 운영 주체, 장소 고정 여부, 강사/팀/호스트 구조 |
| 수익 방식 | 돈이 어디서 발생하는지 | 수강료, 입장료, 멤버십, 대관, 파티 티켓, 워크샵 |
| 장소 생태계 | 전용 공간이 있는지, 여러 장소를 순회하는지 | 반복 venue, Kakao/Naver map 링크, 공간 공식 계정 |
| 강습 구조 | 학원 정규반 위주인지 아마추어/개인 강습 위주인지 | 커리큘럼, 기수제, 원데이, 개인 레슨, 워크샵 |
| 커뮤니티 구조 | 공식 허브가 있는지, 흩어진 개인 계정 중심인지 | 통합 캘린더, 오픈채팅, 링크트리, 카페, 인스타 해시태그 |
| 성장/위험 신호 | 확장 가능성 또는 불안정성 | 신규 모집 빈도, 폐쇄/휴면 계정, 날짜 없는 홍보만 반복 |

각 판단은 반드시 출처와 함께 남긴다. 추측은 `가설`로만 표시하고, 저장 후보나 운영 판단으로 승격하지 않는다.

## 실행 프로필

| 프로필 | 용도 | 저장 |
|---|---|---|
| `swing-daily` | 매일 08:00 스윙 소셜/강습/행사 안정 수집 | 허용 |
| `expanded-research` | 살사/바차타/탱고/스트릿 소스 조사 | 금지 |
| `expanded-ingestion` | 검증 완료된 타장르 후보 수집 | 허용 |

자동 실행 기본값은 반드시 `swing-daily`다. 타장르는 사용자가 명시적으로 요청한 수동 작업에서만 `expanded-research` 또는 `expanded-ingestion`을 쓴다.

## 타장르 조사 기준

1. 공식 원본 우선
   - 인스타그램 공식 계정, 공식 웹사이트, 공식 예약/공지 페이지를 우선한다.
   - 블로그 요약, 뉴스, 홍보 모음 페이지는 원본 탐색 용도로만 사용한다.

2. 일정 단위가 명확해야 함
   - 날짜, 장소, 활동 타입이 확인되어야 한다.
   - 상시 클래스 소개처럼 날짜가 없는 페이지는 저장하지 않는다.

3. 이미지 검증 필수
   - 포스터 또는 대표 이미지가 실제 이벤트를 설명해야 한다.
   - 정사각 크롭/프로필 썸네일/저해상도 이미지는 저장 금지.

4. 모집은 별도 활동 타입
   - 오디션, 참가자 모집, 팀원/크루 모집은 `activity_type: recruit`.
   - 장르 필터와 별개로 모집 정보는 전체 모집 섹션에서 함께 볼 수 있게 설계한다.

## 운영 반영 방식 결정 기준

타장르를 무조건 `/캘린더`와 `강습&행사`에 모두 저장하지 않는다. 이미 해당 장르에서 잘 작동하는 전문 허브가 있으면 Rhythmjoy는 그 허브를 소개하고, 자체 캘린더에는 검증 가치가 높은 항목만 제한적으로 넣는다.

| 반영 방식 | 조건 | 화면 처리 |
|---|---|---|
| `external_hub` | Tango Calendar, DanceCode/FREEZE처럼 해당 씬이 이미 모이는 활성 허브가 있고, 자체 일정 품질이 높음 | 장르 탭 상단에 “전문 일정 허브” 카드로 링크. 자체 이벤트는 주요/검증 항목만 표시 |
| `curated_ingestion` | 허브는 있지만 원본 포스터와 venue 계정을 역추적할 수 있어 우리 검수 가치가 있음 | 관리자 전용 후보로 수집 후 승인된 것만 캘린더/강습&행사 표시 |
| `direct_ingestion` | 전문 허브가 약하거나 정보가 흩어져 있고, 공식 계정에서 날짜/장소/이미지가 반복 확인됨 | 검증된 source만 `expanded-ingestion`으로 저장 |
| `research_only` | 날짜 없는 상시 홍보, 여행/리뷰성 정보, 원본 불명확, 이미지 없음 | 운영 UI 비노출. 씬맵 문서에만 근거로 보관 |

현재 1차 판단:
- 스트릿: DanceCode/FREEZE가 강하므로 `external_hub + curated_ingestion`이 맞다.
- 탱고: Tango Calendar가 강하므로 `external_hub` 우선, 포스터가 있는 페스티벌/워크샵만 `curated_ingestion`.
- 살사/바차타: Latin in Seoul은 현황판 역할이 강하고 venue별 원본이 흩어져 있어 `curated_ingestion`으로 시작한다.

## 장르별 접근 전략

### 살사/바차타
- 소셜바/라틴바 공식 계정과 라틴 통합 일정 사이트를 먼저 확인한다.
- 살사/바차타가 한 포스트에 섞여 있으면 본문 기준으로 `dance_scope`를 재판정한다.
- 소셜, 파티, 클래스, 모집을 분리해서 저장한다.

### 탱고
- 밀롱가/프랙티카/워크숍 구분이 중요하다.
- 캘린더형 사이트는 이미지가 없을 수 있으므로, 이미지 없는 일정은 바로 저장하지 않고 원본 포스터 계정으로 역추적한다.

### 스트릿
- 학원 상시 강좌와 실제 모집/워크숍/배틀을 분리한다.
- 상시 소개 페이지는 저장하지 않는다.
- 날짜가 있는 워크숍, 배틀, 오디션, 참가자 모집만 후보가 된다.

## 저장 승격 조건

`expanded-research`에서 아래가 확인된 소스만 `expanded-ingestion` 저장 대상이 된다.

- 최근 30일 내 실제 일정 게시가 있음
- 날짜/장소/이미지가 안정적으로 추출됨
- 중복 판정 기준이 작동함
- 운영 UI에서 관리자만 확인 가능한 상태로 검수 가능

## 안정화 체크

수집 규칙 변경 후 반드시 실행한다.

```bash
node scripts/test-ingestion-standards.mjs
bash -n scripts/run-ingestion.sh
node --input-type=module -e "import { getAutomationSourceList } from './scripts/ingestion/collection-registry.mjs'; console.log(getAutomationSourceList().every(s => s.scope === 'swing'))"
```

`run-ingestion.sh`는 `source`로 실행하지 않는다. source 실행은 실제 래퍼를 잘못 켤 수 있으므로 스크립트에서 차단한다.
